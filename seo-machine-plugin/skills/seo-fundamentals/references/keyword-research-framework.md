# Keyword Research Framework

## 6-Step Research Brief Development

### Step 1: Seed Keyword Expansion
Start with the primary topic and expand systematically:
- Direct variations (synonyms, plurals, abbreviations)
- Long-tail extensions ("how to [topic]", "[topic] for beginners", "best [topic] tools")
- Question-based queries ("what is [topic]", "why [topic] matters", "how does [topic] work")
- Modifier-based queries ("[topic] examples", "[topic] template", "[topic] checklist")
- Comparison queries ("[topic] vs [alternative]", "[topic] alternatives")

### Step 2: Volume & Difficulty Assessment
For each keyword, assess:
- **Search volume signals**: High (10K+), Medium (1K-10K), Low (<1K)
- **Keyword difficulty**: Easy (<30), Moderate (30-60), Hard (60+)
- **Cost-per-click**: Indicates commercial value (higher CPC = higher business value)
- **Trend direction**: Rising, stable, declining (use Google Trends)

### Step 3: Intent Classification
Classify each keyword:
| Intent | Signals | Content Type |
|--------|---------|-------------|
| Informational | how, what, why, guide, tutorial | Blog post, guide, explainer |
| Commercial | best, top, review, compare, vs | Comparison page, listicle, review |
| Transactional | buy, download, pricing, free trial | Landing page, product page |
| Navigational | brand name, login, specific page | Brand page, feature page |

### Step 4: Competitive Gap Analysis
For the target keyword:
1. Analyze top 10 SERP results
2. Note content length (average, median, range)
3. Identify common topics covered by all competitors
4. Find topics covered by some but not all (opportunities)
5. Identify angles NO competitor covers (differentiation goldmine)
6. Assess SERP features present (featured snippets, PAA, image packs)

### Step 5: Content Architecture Planning
Based on research, recommend:
- **Optimal content length** (beat median of top 10 by 20-30%)
- **Must-cover sections** (topics all competitors cover — table stakes)
- **Differentiator sections** (unique angles, original data, deeper expertise)
- **Featured snippet targets** (format content to capture snippets)
- **Internal linking opportunities** (existing content to connect)

### Step 6: Meta Element Preview
Draft before writing:
- 3 title tag options (50-60 chars each)
- 3 meta description options (150-160 chars each)
- URL slug recommendation
- Primary H1 and H2 outline

## Keyword Mapping Matrix

For each page on the site, maintain a mapping:

| Page URL | Primary Keyword | Secondary Keywords | Intent | Target Position | Current Position | Status |
|----------|----------------|-------------------|--------|----------------|-----------------|--------|
| /page    | keyword        | var1, var2, var3   | Info   | Top 3          | #15             | Optimizing |

### Cannibalization Detection
When multiple pages target the same keyword:
1. Check which page Google ranks (may not be the intended one)
2. Consolidate: merge thin pages into one comprehensive page
3. Differentiate: adjust targeting so each page has unique primary keyword
4. Redirect: 301 redirect the weaker page to the stronger one
5. Canonical: if pages must coexist, set canonical to the preferred version

## Keyword Clustering Algorithm

Group related keywords into clusters for topic coverage:

1. Collect all keyword variations for a topic (50-200 keywords)
2. Group by SERP similarity (keywords with 3+ shared URLs in top 10 = same cluster)
3. Each cluster = one page target
4. Select primary keyword per cluster (highest volume + most relevant)
5. Remaining keywords become secondary targets for that page
6. Map clusters to topic cluster architecture (pillar vs. cluster pages)
