# Topic Cluster Mapping Guide

## Step-by-Step Cluster Building

### Phase 1: Seed Topic Selection

Choose seed topics that meet ALL criteria:
- Core to your business offering
- Sufficient search volume (aggregate cluster volume: 10K+ monthly)
- Room for 8-15+ subtopic pages
- You have expertise or can develop it
- Not dominated by mega-sites for every subtopic

### Phase 2: Subtopic Discovery

For each seed topic, mine subtopics from:

1. **Google Autocomplete** — type seed keyword, note suggestions
2. **People Also Ask** — expand PAA boxes in SERPs
3. **Related Searches** — bottom of SERP page
4. **Keyword tools** — "questions", "topics", "related keywords" reports
5. **Competitor content** — what subtopics do their blogs/guides cover?
6. **Reddit/Quora/Forums** — what questions does the community ask?
7. **Support tickets** — what do customers ask about this topic?

### Phase 3: Cluster Organization

Organize discovered subtopics into a structured cluster:

```
PILLAR: [Broad Topic] (e.g., "Email Marketing")
│
├── CLUSTER: Getting Started
│   ├── What is email marketing?
│   ├── Email marketing for beginners
│   └── How to build an email list
│
├── CLUSTER: Strategy
│   ├── Email marketing strategy guide
│   ├── Email segmentation best practices
│   └── Email personalization techniques
│
├── CLUSTER: Execution
│   ├── Email subject line formulas
│   ├── Email design best practices
│   └── Email automation workflows
│
├── CLUSTER: Metrics
│   ├── Email marketing KPIs
│   ├── How to improve email open rates
│   └── Email deliverability guide
│
└── CLUSTER: Tools
    ├── Best email marketing platforms
    ├── [Tool A] vs [Tool B]
    └── Email marketing templates
```

### Phase 4: Internal Linking Map

For each cluster, define linking relationships:

**Pillar → Cluster links**: The pillar page links to every cluster page using descriptive anchor text related to the subtopic.

**Cluster → Pillar links**: Every cluster page links back to the pillar with anchor text matching the pillar's primary keyword.

**Cluster → Cluster links**: Each cluster page links to 2-3 related cluster pages within the same topic.

**Cross-cluster links**: Occasionally link between different topic clusters when genuinely relevant.

### Phase 5: Content Calendar

Publish in this order for maximum cluster effect:
1. **Pillar page first** — establishes the hub
2. **Highest-value cluster pages** — sorted by search volume × business relevance
3. **Supporting content** — FAQ, glossary, tools pages
4. **Fill gaps** — remaining cluster pages to complete coverage

**Cadence**: 2-4 cluster pages per month per active cluster. Complete a cluster within 2-3 months.

### Phase 6: Performance Monitoring

Track cluster health monthly:
- Total organic traffic to the cluster (all pages combined)
- Number of ranking keywords per cluster
- Internal link integrity (no broken links, all connections intact)
- Content freshness (flag pages >12 months without update)
- Cannibalization check (multiple pages ranking for same keyword)
